# Client Communication

## Status Update Template
```markdown
## Project Update: [Date]

### Completed This Week
- [Feature/task with brief description]
- [Feature/task with brief description]

### In Progress
- [Feature/task + expected completion]

### Blockers/Questions
- [Any blockers needing client input]

### Next Week
- [Planned work]

### Demo Available
[Link to staging/preview if applicable]
```

## Technical Decision Document
```markdown
## Decision: [Title]

### Context
[Why this decision is needed]

### Options Considered
1. **Option A**: [Description]
   - Pros: [...]
   - Cons: [...]

2. **Option B**: [Description]
   - Pros: [...]
   - Cons: [...]

### Recommendation
[Chosen option with rationale]

### Impact
[Timeline, cost, technical debt implications]
```

## Handoff Documentation
- Setup instructions (from zero to running)
- Architecture overview with diagrams
- Environment variables explained
- Common maintenance tasks
- Troubleshooting guide
- Contact info for support

## Estimation Template
```markdown
## Feature: [Name]

### Scope
[What's included / not included]

### Estimate
- Development: X hours
- Testing: Y hours
- Buffer: Z hours
- **Total**: X+Y+Z hours

### Assumptions
[What we're assuming to be true]

### Risks
[What could change the estimate]
```
