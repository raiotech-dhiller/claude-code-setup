# Onboarding Copy

## Onboarding Principles

### Goal
Get users to their first "aha moment" as fast as possible.

### Principles
1. Show value before asking for effort
2. One step at a time
3. Celebrate progress
4. Allow skipping (when possible)
5. Set expectations

## Permission Requests

### Before Asking
Explain the value before requesting access.

```
‚ùå "Allow notifications?"

‚úì "Want to know when teammates respond?
   Turn on notifications to stay in the loop.
   [Allow notifications] [Not now]"
```

### Permission Copy Patterns
```
For notifications:
"Get notified when [specific value].
You can change this anytime in Settings."

For location:
"Allow location to [specific benefit].
We only use this when you [context]."

For contacts:
"Find friends on [Product] by allowing access to your contacts.
We'll never contact them without your permission."
```

## Form/Input Screens

### Progressive Disclosure
Only ask for what's needed now.

```
Screen 1: "What should we call you?" [Name]
Screen 2: "What brings you here?" [Use case selector]
Screen 3: "Invite your team (optional)" [Email inputs]
```

### Input Guidance
```
"What's your company name?"
Helper: This appears on invoices and reports.

"Choose a workspace name"
Helper: Your team will see this. You can change it later.
```

### Optional Fields
```
"Add a profile photo (optional)"
"Invite teammates ‚Äî or do this later"
"Skip this step" / "I'll do this later"
```

## Onboarding Checklist

- [ ] Can users get to value quickly?
- [ ] Is each step clearly explained?
- [ ] Can users skip optional steps?
- [ ] Are expectations set (time, effort)?
- [ ] Is progress visible?
- [ ] Are celebrations appropriate (not over the top)?
- [ ] Is help available at each step?
- [ ] Can users return to onboarding if needed?
